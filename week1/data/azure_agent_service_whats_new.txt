What's New in Foundry Agent Service - Timeline of Features

October 2025:
- Additional model support: gpt-5, gpt-5-mini, gpt-5-nano, gpt-5-chat

September 2025:
- Computer Use tool (preview): A specialized AI tool that can perform tasks by interacting with computer systems and applications through their user interfaces.

August 2025:
- Java SDK Availability: Public preview of the Azure AI Agent Service Java SDK
- Browser Automation tool (Public Preview): Perform real-world browser tasks through natural language prompts in an isolated browser session using Microsoft Playwright Workspaces
- New regional availability: Brazil south, Germany west central, Italy north, South central US

June 2025:
- Deep Research tool: Multi-step research process built on top of the Azure OpenAI o3-deep-research model using Bing Search as knowledge source
- Model Context Protocol (MCP) tool: Extend capabilities of agents by connecting them to tools hosted on remote MCP servers

May 2025 - Agent Service GA:
- Foundry Visual Studio Code extension: Deploy and configure agents natively in VS Code
- Connected Agents: Create task-specific agents that interact seamlessly with a primary agent for multi-agent systems without external orchestrators
- Trace Agents: Debug and monitor agents by tracing agent threads to see inputs and outputs of each primitive
- Trigger agents using Azure Logic Apps: Automatically invoke AI agent when events occur (new email, customer ticket)
- New tools: Bing Custom Search, Morningstar data source

April 2025:
- Azure Monitor integration: Metrics for files indexed, number of runs in timeframe
- BYO Thread Storage: Standard Agent Setup supports Bring Your Own thread storage using Azure Cosmos DB for NoSQL

March 2025:
- Microsoft Fabric tool: Interact with data in Microsoft Fabric through chat for data-driven insights

February 2025:
- Agent Service available in Microsoft Foundry portal: Create, debug, modify agents, view threads, add tools without code

December 2024 - Public Preview Launch:
- Built on Assistants API from Azure OpenAI with additional features
- Additional tools: Bing grounding, function calling
- Non-OpenAI model support: Llama 3.1-70B, Mistral-large-2407, Cohere command R+
- Enterprise security: Keyless authentication, no public egress
- Microsoft-managed storage or bring your own
- SDK support: .NET, Python (Azure SDK), Python (OpenAI SDK)
- Debugging with Application Insights tracing
